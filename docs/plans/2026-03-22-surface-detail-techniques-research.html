Blender 的 Bump、Normal、Displacement 贴图

1.1 Bump Map（凹凸贴图）

原理 低开销 EEVEE 支持

灰度图（8bit，256 级灰度），Bump 节点通过有限差分法计算高度场梯度，构造新的表面法线。不改变实际几何体，仅修改着色法线。

  • 内存占用最小，性能最优
  • 从掠射角观察时容易穿帮（轮廓线仍然平坦）
  • Strength：0.1–1.0，控制凹凸强度
  • Distance：0.01–0.1（场景相关），控制世界空间尺度
  • 适用场景：皮肤毛孔、微小皱纹、砖缝等细微纹理
Image Texture (Non-Color) → Bump Node (Height) → Principled BSDF (Normal)

1.2 Normal Map（法线贴图）

原理 低开销 EEVEE 支持

RGB 三通道分别编码 X/Y/Z 方向的法线偏移（R=X+, G=Y+, B=Z+）。紫蓝色外观来源于大部分法线朝 Z+ 方向（切线空间 0.5, 0.5, 1.0）。通常由高模烘焙到低模获得。

  • 比 Bump Map 更精确，能表达更复杂的表面方向变化
  • 同样不改变几何体，轮廓线不受影响
  • 空间类型：Tangent Space（最常用）/ Object Space / World Space
  • 组合用法：Normal Map 输出 → Bump Node (Normal Input)，实现叠加效果
Image Texture (Non-Color) → Normal Map Node (Color) → Principled BSDF (Normal)

1.3 Displacement Map（置换贴图）

原理 高开销 仅 Cycles

灰度/高度图驱动顶点实际位移，生成真实几何变形。改变真实几何体 → 影响轮廓线、可投射阴影、可遮挡其他物体。

  • Displacement 节点：标量高度输入，沿法线方向位移
  • Vector Displacement 节点：RGB/向量输入，可沿任意方向位移
  • Scale：控制位移幅度（世界空间单位）
  • Midlevel：默认 0.5，黑=最低时设为 0.0
  • 贴图精度：推荐 16/32-bit 浮点 EXR，8-bit 会产生阶梯伪影

Cycles 位移模式

模式几何变形凹凸着色说明
Bump Only（默认）否是即使连接 Displacement 节点也不会有几何变化
Displacement Only是否细分不足时显得块状/棱角分明
Displacement and Bump（推荐）是是大尺度几何位移 + 小尺度凹凸着色，无需极端细分

1.4 自适应细分（Adaptive Subdivision）

底层技术

基于 OpenSubdiv 库，支持 Catmull-Clark 和 Simple 两种细分模式。Blender 4.x 需开启 Experimental；5.0+ 已内置无需切换。

设置步骤：

  1. 添加 Subdivision Surface 修改器 → 启用 Adaptive Subdivision
  2. Shader Editor：Height Texture (Non-Color) → Displacement Node → Material Output.Displacement
  3. 材质设置 Displacement 模式为 "Displacement and Bump"
  4. 调整 Render Properties > Subdivision > Dicing Rate（值越小细节越多，默认 1.0 像素）
  • 全局 Dicing Rate：Render Properties > Subdivision
  • 每对象 Dicing Rate：Subdivision 修改器上可单独设置
  • Dicing Camera：可指定活动相机，使细分基于相机距离
  • 值 0.5 ≈ 0.5 像素微多边形，非常精细
  • vs 手动细分：手动全局均匀加密浪费内存；Adaptive 仅在需要处加密

02

Blender Tissue 插件

2.1 概述

Tissue — 计算设计工具集

作者：Alessandro Zomparelli（Co-de-iT）
定位：面向计算设计的免费开源插件，桥接参数化设计与 Blender 网格建模
核心理念：基于组件的生成式设计 — 以基础网格拓扑驱动组件的复制和变形
状态：Blender 2.79+ 内置（需手动启用）| 最新 v0.3.72（2024.10）

2.2 核心功能

Tessellate（镶嵌/铺砌）

  • 将 Component 复制到 Generator/Base 的每个面上，自动通过重心/双线性插值映射
  • 填充模式：Quad / Tri / Fan / Patch（细分曲面插值，更平滑有机）
  • Thickness：沿面法线拉伸，通过顶点组或均匀值控制
  • Merge：相邻组件边界顶点合并，创建无缝网格
  • 支持 Mesh、Curve、Surface、Text、Meta 等输入类型
  • 迭代镶嵌：输出可作为下一次基础，实现分形递归

Dual Mesh（对偶网格）

  • 三角网格 → 六边形图案；四边网格 → 偏移四边网格
  • 结合 Wireframe + 细分修改器可产生 Voronoi/蜂窝式有机造型

Polyhedral Wireframe（多面体线框）

  • 通过拓扑描述内部多面体定义体积对象
  • 生成清洁拓扑，适合数字制造（3D 打印）
  • 属性：Tissue Node Index、Tissue Edge Index、tissue_is_wireframe、tissue_is_outer

Reaction Diffusion（反应扩散）

Gray-Scott 模型

两种虚拟化学物质（激活剂/抑制剂）在网格表面沿边扩散并反应。可产生斑点、条纹迷宫、蠕虫图案、脉冲点、珊瑚/指纹纹理。直接在 3D 网格拓扑上运算，适用于任意 3D 表面。

  • 控制参数：扩散速率、供给率(f)、杀灭率(k)、迭代次数
  • 权重集成：结果顶点组可驱动其他 Tissue 操作，形成反馈循环
  • 支持各向异性行为，可基于主应力方向定向结构图案

其他工具

  • Convert to Curves：网格边 → 曲线 → 倒角 → 物理线框/晶格模型
  • Weight Tools：Formula / Curvature / Area / Harmonic / Boundary Distance / Colors 互转
  • Lattice Along Surface / UV to Mesh / Random Materials

2.3 典型工作流

  1. 创建/导入基础网格（通常为细分曲面）
  2. 创建组件网格（小型重复单元）
  3. 应用 Tessellate 将组件分布到基础上
  4. 使用 Dual Mesh 或权重绘制细化图案
  5. 可选：运行 Reaction Diffusion 产生有机变体
  6. 转换边为曲线（线框结构）
  7. 应用材质渲染 / 导出制造

2.4 应用领域

领域应用实例
建筑可视化参数化立面板、穿孔屏风（mashrabiya）、空间桁架、测地穹顶
产品设计Voronoi/晶格灯罩、珠宝花纹、鞋底纹路
3D 打印轻量化晶格结构、Voronoi 镂空、互锁瓷砖系统
数字艺术/VFX有机珊瑚/真菌生长、外星/科幻表面细节
纺织模拟锁子甲网格、编织/针织图案近似

2.5 Tissue vs Geometry Nodes

对比项TissueGeometry Nodes
可用性插件（内置需启用）内置（3.0+）
上手难度低中高
性能Python，密集网格慢C++，显著更快
非破坏性半（手动刷新）完全实时
Tessellate原生强项需手动搭建节点
Reaction Diffusion内置无原生
动画支持有限完整集成
未来方向社区维护官方核心方向

结论：两者互补。Tissue 适合快速实现特定计算设计工作流（尤其 Tessellate 和 Reaction Diffusion）；Geometry Nodes 更通用、更快、更适合动画，但学习曲线更陡。

UE5 BumpOffset / 法线贴图及相关技术

3.1 Normal Map

  • UE5 中最基础且开销最小的表面细节方法
  • 格式：默认切线空间（蓝紫色），引擎内部重建 Z 分量，BC5 压缩
  • FlattenNormal：衰减法线强度
  • BlendAngleCorrectedNormals：正确叠加多层法线（RNM 重定向法线映射）
  • Detail Texturing：不同 Tiling 缩放的第二层法线叠加
  • 性能：极低，仅一次额外纹理采样

3.2 Bump Offset（简单视差映射）

单步视差 极低开销

基于高度图和相机视角计算 UV 偏移。单步光线近似 — 无自遮蔽、无轮廓校正。适合浅层深度效果。

  • Height Ratio：偏移强度
  • Reference Plane：0–1，零深度参考面（通常 0.5）
  • 重要规则：所有 PBR 纹理采样必须使用 BumpOffset 输出的偏移 UV

3.3 Iterative Parallax Mapping

  • 将上一次视差 UV 作为下一次输入，多次迭代
  • 法线贴图蓝色通道 × 高度值，防止陡坡纹理游移
  • 性能：每次迭代 +5 shader 指令，通常 ≤4 次

3.4 Parallax Occlusion Mapping (POM)

光线步进视差 高开销 自遮蔽

通过 Ray Marching 穿过高度场找到正确纹理坐标。高度图凸起可遮挡凹陷部分，可选自阴影（第二条光线步进）。

UE5 设置步骤：

  1. 材质编辑器添加 Parallax Occlusion Mapping 材质函数
  2. 连接高度图 Texture Object → Heightmap Texture 输入
  3. 设置 Height Ratio、Min/Max Steps（16–128）
  4. 连接 Pixel Depth Offset 输出
  5. 可选：部分计算移到 CustomizedUVs 优化

优化：LOD 距离衰减 / CameraDepthFade 远距禁用 / 简化材质实例用于远 LOD

3.5 World Position Offset

  • float3 向量加到顶点世界位置，在顶点着色器执行
  • 用途：植被风动、雪面变形、交互水面
  • 注意：不更新碰撞、阴影可能不匹配、Nanite WPO 需 UE 5.4+

3.6 Material Layer 系统

  • 各表面类型作为独立 Material Layer 资产
  • Material Layer Blend 定义混合方式（高度/角度/遮罩）
  • 辅助函数：HeightLerp、MF_ObjectScale、Triplanar Mapping

3.7 技术对比

技术GPU 开销几何变形自遮蔽轮廓线适用场景
Normal Map极低无无无变化所有表面基础细节
Bump Offset低无无无变化远处/浅层深度
Iterative Parallax中无部分无变化中等深度效果
POM高无有 + 自阴影无变化英雄资产近景
World Position Offset低-中顶点级无有变化动态效果
Nanite Displacement中真实几何有有变化大规模场景/地形

3.8 推荐选择指南

目标推荐技术
细粒度表面细节（划痕、颗粒）Normal Map（Detail 层）
浅层深度错觉（砂浆线）BumpOffset
深度表面浮雕（鹅卵石、深裂缝）POM（配合距离衰减）
真实几何位移（静态网格）Nanite Displacement 或烘焙高模 + Nanite
地形细节几何Virtual Heightfield Mesh
动态顶点动画World Position Offset
多材质混合Material Layers + Height Blend
04

UE5 曲面细分（Tessellation）

4.1 历史背景

UE4 → UE5 的范式转变

UE4：DX11 硬件曲面细分（Hull/Domain Shader）+ Tessellation Multiplier + World Displacement
UE5：完全移除，材质编辑器不再暴露相关引脚

移除原因：

  • 与 Nanite GPU 驱动的集群渲染管线不兼容
  • 性能不可预测：平坦过度细分、AMD GPU 尤其昂贵
  • UE5 渲染管线围绕 Nanite + Lumen 重构

4.2 Nanite 核心机制

  • 网格分解为 ~128 三角形集群（Cluster），构建层次化 DAG
  • 每集群独立选择 LOD（屏幕空间误差阈值）
  • 同一网格不同部分可同时以不同 LOD 渲染
  • 小三角形：软件光栅化（Compute Shader）；大三角形：硬件光栅化
  • 完全 GPU 驱动，消除 CPU Draw Call 开销
  • Visibility Buffer 替代传统 G-Buffer
  • 几何数据按需流式加载
工作流范式转变

UE4 思路：低模 + 运行时细分 + 位移贴图 → 生成高精度几何
UE5 思路：DCC 预烘焙高模 → 导入 → Nanite 自动 LOD
从"运行时生成细节"转变为"导入完整细节，运行时智能简化"

4.3 Nanite Tessellation（UE 5.3 → 5.5）

版本状态能力
UE 5.2/5.3实验性主要针对地形，软件光栅化管线内细分
UE 5.4Beta扩展到任意 Nanite 静态网格，实时可调
UE 5.5Beta+性能优化，Lumen GI/反射交互改善

启用方式：

// DefaultEngine.ini
r.Nanite.AllowTessellation=1
r.Nanite.Tessellation=1
  1. 启用 Nanite Displaced Mesh 插件
  2. 材质 Detail 面板启用 Tessellation → 解锁 Displacement 引脚

4.4 替代方案汇总

方案说明局限性
Nanite Tessellation官方推荐，基于高度图位移需 UE 5.3+，5.4 为 Beta
预细分网格DCC 烘焙位移后导入高模文件大，Nanite 处理高效
World Position Offset顶点着色器偏移不增加面数，不更新碰撞
Virtual Heightfield MeshRVT 高度数据生成地形网格仅限地形，GPU 开销大

4.5 Geometry Script

  • UE5 的 Blueprint/Python 程序化几何库（实验/Beta，需手动启用）
  • 操作：ApplyPNTessellation、ApplyUniformTessellation、ApplyMeshBoolean、ApplyDisplaceFromTexture
  • Catmull-Clark 细分：仅编辑器模式
  • Spline Mesh：沿样条线变形静态网格（管道/铁轨/道路）
  • DynamicMeshComponent 不兼容 Nanite（可转 StaticMesh 后启用）

4.6 性能对比：Nanite vs 传统曲面细分

方面UE4 硬件曲面细分UE5 Nanite
三角形生成运行时 Hull/Domain Shader预计算 LOD，运行时选择
LOD 粒度每 Draw Call / Patch每集群（~128 三角形）
过度绘制高极少
CPU 开销中等极低
GPU 开销高密度昂贵，AMD 差随屏幕像素缩放
内存低较高（流式加载）
轮廓质量好优秀
动画支持是（每帧重新细分）有限（5.4+ WPO）
可扩展性高密度时差百亿三角形近恒定开销

关键差异：渲染开销随 屏幕分辨率 缩放而非 场景复杂度，这是 Nanite 的根本架构优势。

4.7 UE 5.4/5.5 新特性

UE 5.4（2024 中）
  • Nanite WPO 支持（有性能代价）
  • Nanite 遮罩材质兼容
  • Nanite Displacement 改进 — 更稳定、更好密度控制
  • Nanite 骨骼网格（实验性）
  • Substrate（前 Strata）材质系统
UE 5.5（2024 末 / 2025 初）
  • Nanite Tessellation 性能优化，减少内存
  • 位移几何正确参与 Lumen GI/反射
  • Virtual Shadow Maps 改进
  • Geometry Script 更多操作、Blueprint 集成改善

5.4 工作流建议

Blender 离线渲染管线
  • 大尺度变形 → Adaptive Subdivision + Displacement（推荐 "Displacement and Bump"）
  • 中等细节 → Normal Map（高模烘焙）+ 可叠加 Bump
  • 微小细节 → Bump Map
  • 复杂镶嵌图案 → Tissue Tessellate
  • 有机/参数化造型 → Tissue（快速原型）+ Geometry Nodes（生产/动画）
  • 16/32-bit EXR 位移贴图获得最佳效果
UE5 实时渲染管线
  • 大尺度变形 → Nanite Tessellation + Displacement（UE 5.4+）
  • 英雄资产近景 → POM（配合 CameraDepthFade）
  • 一般表面深度 → BumpOffset 或 Iterative Parallax
  • 基础细节 → Normal Map（含 Detail 层叠加）
  • 动态效果 → World Position Offset
  • 多材质混合 → Material Layers + Height Blend
Blender → UE5 资产管线
  • Blender Tissue 生成复杂几何 → 导出 FBX/glTF → UE5 启用 Nanite
  • Blender 烘焙 Displacement/Normal Map → UE5 材质中使用
  • Adaptive Subdivision 效果可通过 Nanite Tessellation 近似还原
  • 高模直接导入 + Nanite 是最简单可靠的工作流
